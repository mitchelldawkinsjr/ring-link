# AGENTS.md

## Project Rules

- Use Laravel service/action pattern
- Keep controllers thin
- Use Form Requests for validation
- Use policies for authorization
- Use enums for statuses
- Use feature tests for workflows
- Use Docker for local development
- Use API Resources for responses
- Use transactions for booking workflows
- Avoid premature abstraction
- Maintain pragmatic monolith architecture

## Frontend Rules

- Use TypeScript
- Use Tailwind CSS
- Mobile-first responsive design
- Accessibility compliance
- Use TanStack Query for server state
- Use Zustand for lightweight client state

## Infrastructure Rules

- Use Docker containers
- Redis required for queues
- MySQL 8 required
- Cloudflare R2 for storage
- Frontend deployed to Vercel
- Backend deployed to VPS

## Testing Requirements

- Feature tests required
- API tests required
- CI/CD required
- Linting required
